# Maintenance Rules

**Rule:** Before modifying any file, always study, update, and improve the documentation (`docs` folders) inside the relevant module and theme.

- Review existing documentation to understand the context and architecture.
- Update the documentation with any new changes or insights.
- Consider creating a GitHub Issue or Discussion to track significant changes or solicit feedback.
- Keep the documentation concise, up-to-date, and aligned with the project's conventions.

This rule should be remembered and applied consistently across the codebase.
