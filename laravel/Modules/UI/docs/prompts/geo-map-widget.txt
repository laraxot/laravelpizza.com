bmad-create-story Devi creare un Filament v5 widget chiamato GeoMapWidget all’interno del modulo Geo dell’applicazione Laravel, rispettando rigorosamente l’architettura a moduli e senza inserire codice fuori dal dominio GEO. Il widget deve replicare e migliorare profondamente il comportamento del progetto open source https://github.com/CodeforKarlsruhe/farmshops.eu
 che deve essere studiato a fondo prima di scrivere qualsiasi codice, analizzando in particolare la logica del file direktvermarkter.js e l’intero sistema di gestione dati e mappa. È obbligatorio non inventare funzionalità o librerie inesistenti e non assumere comportamenti non verificati, ma basarsi su standard reali come Leaflet e GeoJSON e verificare sempre le librerie su npm o documentazione ufficiale. Il widget deve mostrare una mappa interattiva basata su Leaflet con caricamento di un unico file GeoJSON contenente massimo circa 3000 punti, caricati una sola volta senza ulteriori chiamate server, replicando il pattern farmshops di dataset statico per performance. Il rendering deve supportare clustering tramite leaflet.markercluster, visualizzazione per zoom con logica LOD (cluster, categoria aggregata, dettaglio), e gestione stateful lato client senza ricaricare dati. Deve includere un sistema di layer multipli (points, cluster, heatmap, zones/poligoni) attivabili tramite UI interna del widget e con possibilità di combinazione layer. Deve includere interazione completa con marker cliccabili che aprono popup, selezione di punti, sincronizzazione con eventuali stati interni e possibilità di filtraggio client-side. Tutta la logica deve essere implementata in un Web Component basato su Lit per isolamento UI, con Leaflet come motore di rendering, senza dipendere da CDN ma solo da npm e build Vite del progetto Laravel. Il widget deve essere altamente modulare, leggibile e scalabile, con separazione tra data layer GeoJSON, rendering layer Leaflet e state management interno. Il sistema deve essere ottimizzato per performance evitando re-render inutili e riducendo al minimo iterazioni su dataset. Deve essere previsto un layer manager interno per gestire attivazione e disattivazione dinamica dei layer. È obbligatorio studiare continuamente le migliori pratiche online su GIS web, Leaflet advanced usage e pattern simili a farmshops, migliorando iterativamente la soluzione. Il lavoro non è considerato completato finché non passa PHPStan, PHPMD, PHP Insights e test Pest senza errori, e non deve contenere codice speculativo o non verificato.
