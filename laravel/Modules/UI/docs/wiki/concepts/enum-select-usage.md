## Overview
`EnumSelect` is a reusable Iron Select for PHP-backed enums in Filament v5.

**Key Features:**
- : Enums via `->enum(MyEnum::class)`
- : Handles HasLabel/HasIcon interfaces
- : Fallback labels/icons via case name
- : HTML support in labels
- : Tom Select compatibility

**Usage Example:**
```php
use Modules\UI\Filament\Forms\Components\EnumSelect;
use App\Enums\TicketTypeEnum;

EnumSelect::make('type_id')
    ->enum(TicketTypeEnum::class)
    ->required()
    ->searchable()
    ->preload();
```

## Icon Support
If your enum implements `Filament\Support\Contracts\HasIcon`, you can enable icons:
```php
EnumSelect::make('status')
    ->enum(StatusEnum::class)
    ->icons()
    ->htmlLabels(); // required for icon HTML rendering
```

## HTML Labels
When you need rich HTML (e.g., SVG icons), enable HTML labels:
```php
EnumSelect::make('priority')
    ->enum(PriorityEnum::class)
    ->icons()
    ->htmlLabels();
```

## Fallbacks
- If the enum does **not** implement `HasLabel`, the component falls back to `$case->label()` if the method exists, otherwise the case name.
- If the enum does **not** implement `HasIcon`, it falls back to `$case->icon()` if defined, otherwise no icon.

## Validation
The component validates that the provided class is a **backed** enum. Invalid classes throw clear exceptions during form building.

## Conversion
Values are stored as the backed enum value. When retrieving the model attribute, Filament automatically casts the value back to the enum case using the component's internal `convertToEnum()` method.
