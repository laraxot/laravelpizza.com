# Consolidamento documentazione

## Obiettivo

Ridurre duplicazioni, normalizzare i nomi e mantenere i link aggiornati.

## Passi operativi

1. Identificare duplicati con suffissi e varianti.
2. Consolidare contenuti sovrapposti in un unico file.
3. Normalizzare naming in kebab-case.
4. Aggiornare i link interni con percorsi relativi.
5. Archiviare i documenti storici necessari.

## Criticita

- Duplicati con contenuti parzialmente diversi.
- Link obsoleti dopo rinomina.

## Punti di forza

- Inventario documenti ampio.
- Regole di naming documentate.

## Punti di debolezza

- Molti file legacy con naming misto.
- Manca un indice unico stabile.

## Colli di bottiglia

- Validazione manuale dei contenuti.
- Aggiornamento dei riferimenti incrociati.

## Come risolverli

- Definire una check-list di consolidamento.
- Aggiornare i link con una revisione sistematica.

## Religione

- Documentazione coerente e parte del prodotto.

## Filosofia

- Un solo punto di verita per ogni tema.

## Politica

- Nessuna rimozione senza consolidamento o archiviazione.

## Output attesi

- Documentazione piu snella e navigabile.
- Riduzione dei duplicati.
