## Come Contribuire

1. Creare un nuovo branch per le modifiche
2. Aggiornare la documentazione esistente o aggiungere nuovi file
3. Assicurarsi che tutti i link siano corretti
4. Inviare una pull request per la revisione

## Convenzioni

- I nomi dei file devono essere in minuscolo con trattini come separatori
- Usare il formato Markdown (.md) per tutti i documenti
- Includere sempre un'intestazione con titolo e descrizione
- Mantenere aggiornato l'indice

## Manutenzione

La documentazione viene aggiornata in concomitanza con le modifiche al codice. Ogni modifica significativa al codice deve essere accompagnata dagli opportuni aggiornamenti alla documentazione.
