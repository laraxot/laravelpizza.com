# Regole per i Collegamenti nella Documentazione

## Principi Base

1. **Bidirezionalità**
   - Ogni collegamento deve essere bidirezionale
   - Se A riferisce B, B deve riferire A
   - Mantenere i collegamenti sincronizzati

2. **Path Relativi**
   - Usare sempre path relativi
   - Facilitare la portabilità
   - Evitare path assoluti

3. **Convenzioni di Naming**
   - Nomi file in kebab-case
   - Estensione .md obbligatoria
   - Nomi descrittivi e chiari

## Gestione dei Collegamenti

### Aggiunta Collegamenti

1. Identificare i moduli correlati
2. Aggiungere collegamenti bidirezionali
3. Verificare la correttezza dei path
4. Testare i collegamenti

### Modifica Collegamenti

1. Identificare tutti i riferimenti
2. Aggiornare path se necessario
3. Mantenere la bidirezionalità
4. Verificare dopo le modifiche

### Rimozione Collegamenti

1. Identificare tutti i riferimenti
2. Rimuovere collegamenti bidirezionali
3. Aggiornare la documentazione
4. Verificare la consistenza

## Verifica Collegamenti

### Manuale

1. Navigare i collegamenti
2. Verificare destinazioni
3. Controllare bidirezionalità
4. Testare anchor

### Automatica

1. Usare link checker
2. Verificare path relativi
3. Controllare file esistenza
4. Validare markdown

## Best Practices

1. **Organizzazione**
   - Mantenere una struttura chiara
   - Raggruppare collegamenti correlati
   - Documentare le relazioni

2. **Manutenzione**
   - Review periodica
   - Aggiornare con le modifiche
   - Rimuovere collegamenti obsoleti

3. **Documentazione**
   - Spiegare le relazioni
   - Documentare le dipendenze
   - Mantenere una mappa aggiornata

## Note Importanti

1. I collegamenti sono parte della documentazione
2. Devono essere mantenuti aggiornati
3. Devono essere verificati regolarmente
4. Devono seguire le convenzioni stabilite

## Checklist Verifica

- [ ] Anchor funzionanti
- [ ] Documentazione aggiornata
- [ ] Mappa collegamenti aggiornata
- [ ] Convenzioni rispettate
