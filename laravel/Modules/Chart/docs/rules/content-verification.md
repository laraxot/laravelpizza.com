# Regole per la Verifica dei Contenuti

## Principi Fondamentali

1. **Verifica Primaria**
   - Cercare sempre il contenuto effettivo prima di rispondere
   - Non basarsi su descrizioni secondarie
   - Verificare le fonti originali

2. **Completezza**
   - Verificare tutto il contenuto, non solo parti
   - Controllare il contesto completo
   - Considerare tutte le versioni disponibili

3. **Tracciabilità**
   - Documentare le fonti utilizzate
   - Mantenere un registro delle verifiche
   - Indicare chiaramente le fonti nelle risposte

## Processo di Verifica

### 1. Identificazione Fonti
- Cercare nei file di configurazione
- Verificare i template
- Controllare i file di traduzione
- Esaminare la documentazione tecnica

### 2. Strumenti di Ricerca
- Utilizzare `grep_search` per contenuti testuali
- Verificare i file di template con `read_file`
- Controllare la struttura con `list_dir`
- Utilizzare `codebase_search` per ricerche semantiche

### 3. Validazione
- Confrontare con la documentazione esistente
- Verificare la coerenza tra le fonti
- Validare il formato e la struttura
- Controllare le dipendenze

## Checklist di Verifica

### Per Contenuti Statici
- [ ] Identificare la fonte primaria
- [ ] Verificare il contenuto effettivo
- [ ] Controllare le traduzioni
- [ ] Validare il formato
- [ ] Verificare le dipendenze

### Per Contenuti Dinamici
- [ ] Identificare la logica di generazione
- [ ] Verificare i parametri di configurazione
- [ ] Controllare le condizioni di visualizzazione
- [ ] Validare i template
- [ ] Verificare le dipendenze

## Best Practices

### 1. Ricerca
- Utilizzare strumenti appropriati per ogni tipo di ricerca
- Verificare più fonti quando possibile
- Documentare il processo di ricerca

### 2. Validazione
- Creare una checklist specifica per ogni tipo di contenuto
- Verificare la coerenza tra le fonti
- Validare il formato e la struttura

### 3. Documentazione
- Mantenere aggiornata la documentazione
- Creare un indice dei contenuti
- Documentare le fonti di verifica

## Implementazione

### 1. Struttura
- Creare una directory dedicata per la documentazione dei contenuti
- Organizzare i file per tipo di contenuto
- Mantenere un indice aggiornato

### 2. Processo
- Seguire la checklist di verifica
- Documentare ogni passo
- Validare i risultati

### 3. Monitoraggio
- Revisione periodica delle verifiche
- Analisi degli errori
- Aggiornamento delle procedure
