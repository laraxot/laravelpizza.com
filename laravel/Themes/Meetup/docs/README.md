# Meetup Theme Docs

Benvenuto nella documentazione del **Tema Meetup**. Manteniamo tutte le guide organizzate per aree chiave:

1. **Indice principale** → consulta `00-index.md` per una panoramica delle regole critiche del tema.
2. **Architettura Folio/V0lt** → la cartella `02-architecture/` descrive layout condivisi, componenti `x-page` e la regola JSON-only per le pagine pubbliche (`folio-pages-json-only-rule.md`).
3. **Workflow di sviluppo** → in `03-development/` trovi linee guida su npm, Vite e sincronizzazione asset; rileggi `theme-development-rules.md` prima di modificare Blade o CSS.
4. **Componenti & Blocchi** → la cartella `04-features/` contiene piani d’implementazione per hero, eventi e dashboard (es. `homepage-block-implementation-plan.md`).
5. **Troubleshooting** → usa `07-troubleshooting/` per analizzare errori tipici (metatags, namespace pub_theme, vite build output).

> 🔁 Prima di lavorare sul tema aggiorna sempre i file in `docs/` con scoperte, fix e regole. Evita nuovi Markdown fuori da questa cartella e mantieni i nomi in minuscolo (eccetto README.md/CHANGELOG.md).
