# Animation Usage Guidelines - Meetup Theme

## Core Principles

- **Purposeful**: Animations should serve a purpose, guiding the user's attention, providing feedback, or improving perceived performance. Avoid gratuitous animations.
- **Subtle**: Keep animations subtle and fluid to enhance the user experience without being distracting or overwhelming.
- **Performance**: Ensure animations are performant and do not negatively impact the application's responsiveness or load times.

## Implementation Guidelines

- **Transitions**: Use CSS transitions for smooth changes in state (e.g., hover effects, accordion toggles).
- **Duration**: Keep transition durations short (e.g., 150ms-300ms) for a snappy feel.
- **Easing**: Use appropriate easing functions (e.g., `ease-in-out`) for natural-looking animations.
- **Feedback**: Use animations to provide feedback on user actions (e.g., button clicks, form submissions).
- **Micro-interactions**: Incorporate small, delightful animations for micro-interactions (e.g., loading spinners, success checks).

## Examples

- **Hover Effects**: Smooth color changes or slight scaling on interactive elements.
- **Accordion Toggle**: Gentle slide up/down animation when expanding/collapsing content.
- **Form Submission**: Subtle loading spinner on the submit button, followed by a success/error state animation.
