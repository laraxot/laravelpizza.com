# Docs-First Workflow (Meetup Theme)

## Regola locale

Prima di modificare view/component/layout del tema Meetup:

1. studiare la docs del tema (`00-index.md` e guide rilevanti);
2. aggiornare almeno un documento in `Themes/Meetup/docs`;
3. allineare regole globali (`docs/rules`, `docs/memory`, `docs/skills`) se cambia il metodo operativo;
4. valutare aggiornamento/creazione Issue e Discussion GitHub;
5. solo dopo applicare modifiche al codice.

## Focus operativo

- Evitare regressioni runtime (`view:cache`, risoluzione componenti Blade).
- Mantenere la documentazione di troubleshooting aggiornata insieme ai fix.
