# Events Page Implementation

## Overview
The events page at `/it/events` is dynamically generated using a Block component and configured via JSON.

## Configuration
The configuration is located in `laravel/config/local/laravelpizza/database/content/pages/events.json`.

```json
{
    "slug": "events",
    "content_blocks": {
        "it": [
            {
                "type": "events",
                "data": {
                    "view": "pub_theme::components.blocks.events.list",
                    "query": {
                        "model": "Modules\\Meetup\\Models\\Event",
                        "scope": "upcoming",
                        "orderBy": "start_date",
                        "direction": "asc",
                        "limit": 50
                    }
                }
            }
        ]
    }
}
```

## Component
The component `laravel/Themes/Meetup/resources/views/components/blocks/events/list.blade.php` handles the display.
It supports:
- **Dynamic Data**: Fetches data based on the `query` configuration in JSON.
- **Filtering**: Client-side filtering (Alpine.js) for All/Upcoming/Past (visual only, data is limited by query).
- **SEO**: Uses SEO-friendly URLs generated by `Event::toBlockArray()`.

## Model Integration
The `Modules\Meetup\Models\Event` model provides:
- **Scopes**: `upcoming`, `past`, `ordered`.
- **Transformation**: `toBlockArray()` prepares data for the view, including locale-aware URLs.
