## The Correction

### Incorrect Understanding (Before)
I thought the copy command should use `./resources/html/dist/*` as source.

### Correct Understanding (After)
The copy command MUST use `./public/*` as source.

```json
{
  "scripts": {
    "build": "vite build",
    "copy": "cp -r ./public/* ../../../public_html/themes/Meetup"
  }
}
```

## Why `./public/*` is Correct

### 1. Vite Configuration

**File**: `Themes/Meetup/vite.config.js`

**Line 21**:
```javascript
build: {
    outDir: './public',  // ← Vite outputs here!
}
```

### 2. Build Flow

```
SOURCE → BUILD → DEPLOY

resources/css/app.css    →    npm run build    →    public/assets/app-[hash].css
resources/js/app.js                                 public/assets/app-[hash].js
                                                    public/manifest.json

                         →    npm run copy     →    public_html/themes/Meetup/
```

### 3. Standard Convention

- `public/` represents production-ready, publicly accessible files
- Follows standard web server conventions
- Separates source (`resources/`) from build output (`public/`)
- The `resources/html/dist/` directory is NOT used in this theme's build process

### 4. laravel-vite-plugin Behavior

The `laravel-vite-plugin`:
1. Reads from `resources/css/app.css` and `resources/js/app.js` (configured in `input`)
2. Compiles, minifies, and hashes the files
3. Outputs to `./public/` (configured in `outDir`)
4. Generates `manifest.json` in the same directory

## Lessons Learned

1. **Always check `vite.config.js`** to understand where Vite outputs files
2. **`outDir` setting is definitive** - it tells exactly where build files go
3. **Don't assume directory structure** - always verify with actual configuration
4. **`laravel-vite-plugin` uses `public/`** as output by convention
5. **Copy source must match Vite output** - they are coupled by design
