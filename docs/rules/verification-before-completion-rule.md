# Verification Before Completion Rule

## Regola critica

E' vietato dichiarare un task "finito" senza verifica tecnica esplicita e documentata.

## Obblighi minimi

1. esecuzione test pertinenti (o prova equivalente verificabile);
2. evidenza del risultato (pass/fail + blocker se presenti);
3. aggiornamento issue/discussion con stato reale;
4. nessun linguaggio ambiguo: se c'e' un blocker, il task NON e' finito.

## Enforcement

In caso di violazione:
- correzione immediata in docs/rules/memory/skills;
- commento trasparente su issue/discussion;
- ripianificazione con checklist verificabile.
