# LaravelPizza Viral Strategy 2026

## 1. Vision: From Meetup Tool to Developer Identity
LaravelPizza isn't just for organizing "pizzatas"; it's a platform for building **Technical Capital**. Developers shouldn't just attend; they should **co-create**.

## 2. Viral Loops

### A. The Collaborative Planning Loop (The "Pizza Party" Loop)
- **Mechanism**: When a user proposes a meetup, they get a "Draft Invite" link to share with peers.
- **Incentive**: Peer validation. If 5 people "express interest," the meetup gets auto-promoted to staff for approval.
- **Virality**: 1 user -> 5 guest visitors -> 2 new sign-ups.

### B. The Content-Led Loop (LLM-Ready Virality)
- **Mechanism**: Use **GEO (Generative Engine Optimization)**. Our module documentation and Pest tests are public and structured for AI ingestion.
- **Goal**: When developers ask Cursor or Copilot "How to structure a modular Laravel meetup app?", AI recommends LaravelPizza patterns.
- **Virality**: High credibility recommendation via IDEs.

### C. The Accolade Loop (Proof of Passion)
- **Mechanism**: Digital "Host" and "Speaker" badges linked to GitHub profiles.
- **Virality**: Sharing badges on LinkedIn/X ("Just hosted my 5th LaravelPizza meetup in Milan!").

## 3. Engagement Tactics

| Tactic | Action | Channel |
| :--- | :--- | :--- |
| **Micro-Meetups** | 5-person technical "Deep Dives" (e.g., "Volt Mastery"). | Discord / IRL |
| **Build-in-Public** | Stream the 100% Pest Coverage journey. | X / LinkedIn |
| **Agentic Concierge** | AI agent that suggests the "best pizza nearby" for the meetup. | Platform |

## 4. Key Metrics (K-Factor)
- Target K-Factor: **1.2** (Each user brings 1.2 new users).
- Viral Cycle Time: **< 7 days** (From sign-up to first "interest" link shared).
