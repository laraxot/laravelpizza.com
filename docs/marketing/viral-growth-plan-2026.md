# Viral Growth Plan 2026 - LaravelPizza

## Obiettivo

Rendere LaravelPizza un brand/community virale con un growth engine ripetibile basato su meetup, UGC e referral.

## Tesi strategica

La viralita' non e' un singolo contenuto, ma un sistema:

1. eventi ricorrenti ad alto valore;
2. produzione UGC guidata dalla community;
3. referral loop con incentivi;
4. distribuzione creator-native;
5. retention loop post-evento.

## North Star + KPI

North Star: `Active Community Participants / 28 giorni`.

KPI principali:
- RSVP -> show-up rate
- attendee -> UGC rate entro 72h
- attendee -> referral invite rate
- referral invite -> RSVP conversion
- repeat attendance 30/60/90 giorni

## 4 Growth Loops

### Loop A - Event -> UGC -> Discovery

1. evento (pizzata/meetup) con format replicabile
2. kit UGC (template post + hashtag + prompt foto/video)
3. repost ufficiale + leaderboard creator
4. nuovi utenti scoprono -> iscrivono -> partecipano

### Loop B - Event -> Referral -> RSVP

1. ogni attendee riceve codice invito personale
2. reward doppio lato (chi invita + invitato)
3. waitlist dinamica e reminders
4. piu' RSVP qualificati

### Loop C - Expert Content -> Trust -> Attendance

1. speaker/ospite pubblica mini-teaser
2. clipping short-form pre-evento
3. CTA unica verso landing evento
4. trust + conversion registrazioni

### Loop D - Post Event -> Retention -> New Hosts

1. follow-up in 24h (recap + gallery + next date)
2. survey/NPS in 48h
3. recruitment micro-host locali
4. nuove pizzate locali (capillarita')

## Piano 90 giorni

### Fase 1 (Giorni 1-30) - Foundation

- definire 3 format evento (beginner, builder, founder)
- creare media kit UGC standard
- implementare referral base e tracking UTM
- definire dashboard KPI settimanale

### Fase 2 (Giorni 31-60) - Acceleration

- 2 eventi/settimana in citta' core
- partnership con 10 micro-creator verticali
- lancio challenge community mensile
- test pricing/no-show mitigation (free+deposit, paid-lite)

### Fase 3 (Giorni 61-90) - Scale

- playbook host locale + ambassador program
- espansione in nuove citta' con host certificati
- automazione reminder + post-event funnel
- sprint PR/community con storie best-performing

## Esperimenti prioritari

1. CTA RSVP A/B: "Posti limitati" vs "porta un amico"
2. reminder cadence: T-48h/T-6h/T-1h
3. referral incentive: perk digitale vs consumo onsite
4. format evento: workshop pratico vs networking aperto
5. post-event asset: recap video 45s vs foto carousel

## Governance Community

- policy anti-spam e anti-toxic chiare
- moderation handbook (Discord/Community)
- escalation rapida per comportamenti non conformi

## Stack operativo

- Tracking: UTM + event analytics + cohort retention
- Content ops: calendario editoriale settimanale
- Community ops: host roster + ambassador CRM
- Experiment ops: backlog e review bisettimanale

## Definition of Done

Il piano e' in controllo quando:
- RSVP->attendee >= target interno stabile
- attendee->UGC e attendee->referral crescono per 4 settimane consecutive
- repeat attendance 60d in crescita
- ogni evento produce asset riusabile e distributivo
