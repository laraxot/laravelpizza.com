# Events Page Implementation

## Overview
The events page at `/it/events` is dynamically generated using a Block component and configured via JSON.

## Configuration
The configuration is located in `laravel/config/local/laravelpizza/database/content/pages/events.json`.

```json
{
    "slug": "events",
    "content_blocks": {
        "it": [
            {
                "type": "events",
                "data": {
                    "view": "pub_theme::components.blocks.events.list",
                    "title": "Upcoming Events",
                    "description": "Join us for pizza and Laravel discussions",
                    "query": {
                        "model": "Modules\\Meetup\\Models\\Event",
                        "orderBy": "start_date",
                        "direction": "asc",
                        "limit": 50,
                        "wrap_in": "events"
                    }
                }
            }
        ]
    }
}
```

**Note:** No `scope` is specified because Alpine.js handles client-side filtering (All/Upcoming/Past). Setting `scope: "upcoming"` would prevent the "Past Events" filter from showing results.

## Component
The component `laravel/Themes/Meetup/resources/views/components/blocks/events/list.blade.php` handles the display.
It supports:
- **Dynamic Data**: Fetches data based on the `query` configuration in JSON (model, orderBy, direction, limit).
- **Filtering**: Client-side Alpine.js filtering for All/Upcoming/Past using the `status` field from `toBlockArray()`.
- **SEO**: Uses SEO-friendly slug-based URLs generated by `Event::toBlockArray()` with `LaravelLocalization::localizeUrl()`.

## Event Detail Page
The detail page uses Folio model binding:
- **File**: `pages/events/[.Modules.Meetup.Models.Event].blade.php`
- **Route**: `/it/events/{slug}` — resolved by `Event::getRouteKeyName()` returning `'slug'`
- **Component**: `components/blocks/events/detail.blade.php` — full event info with Schema.org JSON-LD

## Model Integration
The `Modules\Meetup\Models\Event` model provides:
- **Scopes**: `upcoming()`, `past()` for server-side filtering when needed.
- **Transformation**: `toBlockArray()` prepares data for the view, including localized URLs via `LaravelLocalization::localizeUrl()`.
- **Schema.org**: `toSchemaOrg()` generates JSON-LD structured data for SEO.
- **Null Safety**: Handles null `start_date` with `Carbon::now()` fallback.
